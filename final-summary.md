# 이약뭐지 최종 요약

작성일: 2026-05-23

## 현재 상태

백엔드 MVP는 원격 Supabase 기준으로 주요 흐름이 동작한다.

검증된 흐름:

```text
Auth
OCR
약품 분석
공공 의약품 DB cache-aside
현재 복용약 등록
복약 일정 생성
체크리스트
복용 로그
리포트
챗봇
상호작용 안전 가드
DUR batch 적재
```

## 가장 괜찮은 부분

- DB 구조는 MVP 기준 충분하다.
- 약 정보는 사전 적재와 OCR 중 cache-aside 적재가 모두 가능하다.
- DUR은 최신 `DURPrdlstInfoService03/getUsjntTabooInfoList03` 기준으로 수정됐고, 적재된 `medications.item_seq` 기준 batch 동기화가 가능하다.
- Gemini는 상호작용 근거가 부족하면 추측하지 않고 답변을 차단한다.
- 복약 일정, 체크리스트, 로그, 리포트의 기본 백엔드 흐름이 원격에서 통과했다.
- `no_registered_warning`을 “안전함”으로 단정하지 않는 정책이 들어갔다.

## 가장 부족한 부분

- 실제 처방전/약봉투 OCR 품질 검증이 부족하다.
- 전체 약품 마스터와 전체 DUR 적재가 아직 완료된 것은 아니다.
- 정기 스케줄러가 실제 운영에 등록된 상태는 아니다.
- FCM은 dry-run 수준이며 실제 단말 수신 검증이 필요하다.
- 챗봇 안전성은 좋아졌지만 질문 테스트셋과 약사/전문가 리뷰가 필요하다.
- 법무/개인정보/의료기기 소프트웨어 검토가 아직 필요하다.
- 운영 모니터링과 실패 알림이 부족하다.

## 다음에 해야 할 일

우선순위는 다음이다.

```text
1. sync-drug-master 정기 적재
2. sync-dur-interactions syncKnownMedications=true batch 전체 실행
3. scheduled job 등록
4. 실제 OCR 테스트셋 구축
5. 프론트에서 OCR → analyze → confirm 흐름 구현
6. 상호작용 UI에서 안전 단정 금지
7. FCM 실제 단말 수신 검증
8. Gemini 안전 regression test 구축
9. 운영 대시보드/실패 알림 구현
10. 의료/법무 검토
```

## 운영 원칙

가장 중요한 원칙은 아래다.

```text
모르면 답하지 않는다.
DB에 등록된 경고가 없다는 말은 안전하다는 뜻이 아니다.
사용자가 확인하지 않은 약은 자동 등록하지 않는다.
복용량 변경, 중단, 대체 복용은 안내하지 않는다.
AI 답변은 항상 참고용이며 의사 또는 약사 확인을 권한다.
```

## 최종 판단

현재 구현은 백엔드 MVP로는 꽤 좋다.

하지만 지금 당장 일반 사용자에게 “안전한 복약 서비스”로 공개하기에는 부족하다. 실제 출시 전에는 OCR 품질, 데이터 적재, scheduled job, FCM 실기기, 챗봇 안전 테스트, 의료/법무 검토를 반드시 끝내야 한다.

따라서 현재 단계는 다음으로 정의한다.

```text
백엔드 MVP 기능 검증 완료
운영/품질/법무 준비 진행 필요
```
